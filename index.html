## Secrets Management

Never store secrets in .env files. Use macOS Keychain instead.

**To store a secret:**
security add-generic-password -a "SECRET_NAME" -s "my-secrets" -w "value"

**To use secrets in commands:**
SECRET_NAME=$(security find-generic-password -a "SECRET_NAME" -s "my-secrets" -w) npm run dev

**When I need secrets:**
1. Check if .secrets.json exists (lists what secrets the project needs)
2. Suggest the keychain command pattern above
3. Never ask me to create .env files or export secrets in shell

**If a secret is missing:**
Tell me to add it: security add-generic-password -a "NAME" -s "my-secrets" -w

That's it. Claude Code will use Keychain commands instead of .env files.

The raw commands (for the curious)

Here's exactly what's happening under the hood:

# Store a secret in macOS Keychain:
security add-generic-password -a "MY_API_KEY" -s "secret-lover" -w "your-secret-value"

# Retrieve a secret (will prompt for password/Touch ID):
security find-generic-password -a "MY_API_KEY" -s "secret-lover" -w

# Run a command with secrets as env vars:
MY_API_KEY=$(security find-generic-password -a "MY_API_KEY" -s "secret-lover" -w) npm run dev

# List all your secrets:
security dump-keychain | grep -A4 'svce.*secret-lover' | grep acct

# Delete a secret:
security delete-generic-password -a "MY_API_KEY" -s "secret-lover"

secret-lover just wraps these with nicer UX. The real security is Apple's.
